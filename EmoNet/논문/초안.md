# 📄 EmoNet: 감정 동역학 기반 중간표현과 스타일 회귀를 이용한 한국어 응답 생성 파이프라인

---

# 1. 서론 (Introduction)

감정 대화 생성은 단순한 자연어 생성 문제가 아니다. 입력 문장의 의미를 이해하는 것만으로는 충분하지 않으며, 그 문장이 담고 있는 정서 상태를 어떤 내부 표현으로 변환하고, 이를 어떤 말투와 표현 방식으로 외부에 드러낼 것인가까지 포함하는 문제이다.

기존의 감정 대화 시스템은 크게 두 가지 접근을 따른다. 첫째는 입력 텍스트로부터 직접 응답을 생성하는 end-to-end 생성 방식이다. 둘째는 감정 라벨을 보조 입력으로 사용하여 생성 모델을 조건부로 제어하는 방식이다. 그러나 두 방식 모두 공통적으로 감정 상태 표현과 언어 생성이 하나의 모델 내부에서 동시에 처리된다는 특징을 갖는다. 이로 인해 다음과 같은 문제가 발생한다.

- 감정 표현이 내부에서 어떻게 형성되는지 해석하기 어렵다
    
- 특정 감정 상태에서 왜 특정 말투가 생성되는지 추적하기 어렵다
    
- 감정과 스타일을 독립적으로 제어하기 어렵다
    

본 연구는 이러한 문제를 해결하기 위해 감정 응답 생성을 다음과 같은 다단계 구조로 분해한다.

```
text → stim_vec → graph dynamics → dominant branch → z → s → response
```

이 구조의 핵심 아이디어는 다음과 같다.

- 감정 상태를 직접 텍스트로 변환하지 않고, 중간 표현을 거치게 한다
    
- 감정의 동적 변화를 그래프 기반 구조로 모사한다
    
- 최종 응답 생성 이전에 스타일 벡터를 명시적으로 분리한다
    

이를 통해 감정 표현과 응답 스타일을 분리하고, 각 단계의 동작을 독립적으로 분석할 수 있는 구조를 만든다.

본 연구의 기여는 다음과 같이 정리된다.

1. 감정 응답 생성을 중간 표현 기반 파이프라인으로 분해한 구조를 제안한다
    
2. 신경조절 개념을 차용한 그래프 동역학 기반 감정 표현 모델을 구현한다
    
3. 로컬 LLM을 활용하여 `(z, s)` 스타일 데이터셋을 자동 구축하는 방법을 제안한다
    
4. 현재 시스템에서 발생하는 스타일 편향을 구조적으로 분석한다

# 2. 관련 연구 (Related Work)

## 2.1 감정 대화 생성

감정 대화 생성은 EmpatheticDialogues와 같은 데이터셋을 기반으로 활발히 연구되어 왔다. 기존 접근은 주로 encoder-decoder 구조에서 감정 라벨을 추가 입력으로 사용하는 방식이다. 그러나 이러한 방식은 감정 정보를 discrete label로 단순화하며, 감정의 연속성과 동적 변화를 충분히 반영하지 못한다.

## 2.2 스타일 제어 텍스트 생성

스타일 제어 생성은 politeness, formality, sentiment 등을 조절하는 연구가 존재한다. 대표적으로 attribute conditioning, prefix tuning, control tokens 등이 사용된다. 그러나 대부분의 방법은 스타일을 직접 텍스트 생성 모델에 주입하는 방식이며, 스타일 자체를 별도의 예측 대상로 분리하지 않는다.

## 2.3 controllable generation

최근 controllable generation에서는 latent variable을 활용하여 생성 과정을 제어하는 연구가 증가하고 있다. 그러나 latent space는 일반적으로 해석 가능성이 낮고, 감정과 스타일을 명확히 분리하지 않는다.

## 2.4 neuro-inspired 모델

신경망 기반 구조에서 biological inspiration을 차용한 연구는 존재하지만, 감정 표현을 그래프 동역학으로 모델링하는 접근은 제한적이다. 본 연구는 생물학적 정확성을 목표로 하지 않고, 감정 상태의 변화를 표현하기 위한 구조적 비유를 사용한다.

# 3. 전체 시스템 개요 (System Overview)

EmoNet은 감정 응답 생성을 하나의 모델이 아니라 다단계 파이프라인으로 구성한다.

전체 흐름은 다음과 같다.

1. 입력 텍스트를 정서 자극 벡터로 변환
    
2. 그래프 동역학을 통해 감정 상태의 흐름 생성
    
3. dominant branch 추출
    
4. branch를 잠재벡터 `z`로 압축
    
5. `z`로부터 스타일 벡터 `s`를 회귀
    
6. 스타일 조건을 반영한 응답 생성
    

이 구조의 핵심 특징은 다음과 같다.

- 감정 상태를 동적 구조로 표현한다
    
- 스타일을 독립 변수로 분리한다
    
- 각 단계가 독립적으로 검증 가능하다
    

---

# 4. 방법 (Method)

## 4.1 텍스트 → 자극 벡터

입력 텍스트는 4차원 자극 벡터로 변환된다.

- dopamine (보상, 동기)
    
- serotonin (안정, 만족)
    
- norepinephrine (긴장, 각성)
    
- melatonin (피로, 휴식)
    

이 변환은 TF-IDF 기반 feature와 Ridge regression을 이용한다. 학습 타깃은 실제 생리 신호가 아니라, 감정 키워드 기반 proxy이다.

따라서 이 단계는 생리적 예측이 아니라, 텍스트의 정서적 특성을 저차원 공간으로 투영하는 과정이다.

---

## 4.2 신경 그래프 동역학

그래프는 256개의 뉴런으로 구성된다.

- inhibitory: 억제
    
- excitatory: 활성화
    
- modulatory: 전역 조정
    

각 뉴런은 다음 상태를 가진다.

- activation (K)
    
- threshold
    
- memory
    
- refractory
    
- dropout 상태
    

tick 단위로 다음 과정이 반복된다.

- 입력 신호 집계
    
- 활성도 업데이트
    
- memory 기반 조정
    
- neuron type별 효과 적용
    
- edge propagation
    
- pruning / rewiring
    

이 과정은 감정 상태가 시간에 따라 변화하는 것을 모사한다.

---

## 4.3 Branch 추출

그래프에서 생성된 활성 경로를 추적하여 branch를 구성한다. 이후 점수 기반으로 상위 경로를 통합하여 dominant branch를 생성한다.

이 branch는 감정 상태의 흐름을 나타내는 구조로 해석할 수 있다.

---

## 4.4 Branch → z

branch는 각 step마다 6차원 feature로 표현된다. 이를 다음 통계량으로 요약한다.

- 평균
    
- 표준편차
    
- 최소 / 최대
    
- 시작 / 끝 값
    
- 변화량
    
- 기울기
    

이후 선형 변환을 통해 64차원 벡터 `z`를 생성한다.

현재 방식은 학습 기반이 아니라 고정 투영이다.

---

## 4.5 z → s

`z`로부터 32차원 스타일 벡터를 예측한다.

스타일 축은 다음과 같다.

- warmth
    
- politeness
    
- cooperativeness
    
- calmness
    
- softness
    
- sharpness
    
- tension
    
- seriousness
    
- directness
    
- verbosity
    

회귀기는 Ridge regression이다.

---

## 4.6 스타일 기반 응답 생성

최종 단계에서 다음 정보를 LLM에 입력한다.

- 원문 텍스트
    
- style vector
    
- style summary
    
- style tags
    

이를 통해 동일한 내용이라도 다른 스타일로 응답을 생성할 수 있다.
