1. 서론  
    1.1 연구 배경  
    한국어 감정 응답 생성은 의미 전달뿐 아니라 정서 상태와 말투의 조절이 중요하다는 점을 설명한다.  
    1.2 문제 제기  
    기존 end-to-end 감정 대화 생성 방식은 감정 형성과 스타일 표면화가 한 모델 안에 섞여 있어, 중간 과정과 병목을 분석하기 어렵다는 한계를 제시한다.  
    1.3 연구 목표  
    감정 자극, 감정 동역학, 잠재표현, 스타일 벡터, 최종 응답 생성 단계를 분리한 모듈형 파이프라인을 제안하는 것을 목표로 한다.  
    1.4 연구 기여  
    EmoNet 파이프라인 제안, 그래프 기반 감정 동역학 구현, 로컬 LLM 기반 스타일 약지도 데이터 구축, 구조적 병목 분석 가능성 제시를 연구 기여로 정리한다.
    
2. 관련 연구  
    2.1 감정 대화 생성  
    감정 조건부 응답 생성과 공감형 대화 생성 연구를 검토한다.  
    2.2 스타일 제어 텍스트 생성  
    politeness, formality, sentiment 등 스타일 속성 제어 연구를 정리한다.  
    2.3 잠재공간 기반 controllable generation  
    잠재벡터를 활용한 생성 제어와 해석 가능성 문제를 다룬다.  
    2.4 neuro-inspired 표현 모델  
    생물학적 비유를 활용한 표현 모델과 본 연구의 차별점을 설명한다.
    
3. 연구 문제 및 시스템 개요  
    3.1 연구 문제  
    감정 응답 생성을 중간표현 기반 파이프라인으로 분해할 수 있는지, z -> s 학습이 가능한지, 병목을 구조적으로 진단할 수 있는지를 질문으로 설정한다.  
    3.2 전체 시스템 개요  
    text -> stim_vec -> graph dynamics -> dominant branch -> z -> s -> response의 전체 흐름을 제시한다.  
    3.3 설계 원리  
    감정 상태와 스타일을 분리하여 해석 가능성과 제어 가능성을 높이려는 설계 의도를 설명한다.
    
4. 제안 방법  
    4.1 입력 텍스트 전처리  
    한국어 대화 데이터를 정규화하고 모델 입력 형식으로 변환하는 과정을 설명한다.  
    4.2 자극 벡터 인코딩  
    입력 텍스트를 dopamine, serotonin, norepinephrine, melatonin의 4차원 정서 자극 벡터로 투영하는 방식을 기술한다.  
    4.3 그래프 기반 감정 동역학  
    256개 뉴런으로 구성된 그래프에서 감정 상태가 확산, 억제, 수렴되는 과정을 설명한다.  
    4.4 branch 추출  
    tick별 활성 경로를 기록하고 dominant branch를 복원하는 절차를 다룬다.  
    4.5 잠재벡터 z 생성  
    dominant branch를 통계 요약 기반으로 64차원 잠재벡터 z로 인코딩하는 과정을 설명한다.  
    4.6 스타일 벡터 s 회귀  
    32차원 스타일 벡터를 예측하는 z -> s 선형 회귀기의 구조와 목적을 기술한다.  
    4.7 스타일 조건부 응답 생성  
    예측된 스타일 벡터와 원문 입력을 바탕으로 로컬 LLM이 최종 응답을 생성하는 과정을 설명한다.  
    4.8 로컬 LLM 기반 스타일 라벨링  
    (z, s) 약지도 데이터셋 구축 절차와 consistency filtering 방식을 기술한다.
    
5. 데이터 및 실험 설정  
    5.1 데이터셋 구성  
    감성 대화 말뭉치 기반 학습 데이터와 실험용 subset 구성을 설명한다.  
    5.2 z 데이터 구축  
    총 51,628개 샘플과 60개 감정 라벨로 구성된 z 데이터 생성 절차를 제시한다.  
    5.3 스타일 라벨링 subset  
    500개 균형 subset 중 200개 샘플을 라벨링 대상으로 사용한 설정을 설명한다.  
    5.4 구현 환경  
    Python CLI, 로컬 LLM, Ollama 호환 엔드포인트, 사용 모델과 주요 실행 명령을 기술한다.  
    5.5 평가 지표  
    consistency L1, keep rate, MAE, baseline 비교, 정성 평가 기준을 설명한다.
    
6. 실험 결과  
    6.1 파이프라인 동작 검증  
    text -> z -> s -> response 경로가 end-to-end로 정상 동작함을 확인한다.  
    6.2 스타일 라벨링 품질  
    200개 중 190개 샘플이 filtering을 통과했고 평균 consistency L1이 0.0998임을 제시한다.  
    6.3 z -> s 회귀 성능  
    평균 MAE 0.1422로 mean baseline 0.1371을 넘지 못한 결과를 보고한다.  
    6.4 스타일 분포 편향  
    warmth, calmness, cooperativeness, softness가 높고 sharpness, tension, dominance가 낮은 편향을 분석한다.  
    6.5 정성적 사례  
    부정적 입력에도 응답이 지나치게 협조적이고 완화된 톤으로 생성되는 예시를 제시한다.
    
7. 분석 및 논의  
    7.1 구조적 병목 분석  
    현재 성능 한계가 회귀기 자체보다 데이터 구성과 스타일 공간 설계에 더 크게 기인함을 논의한다.  
    7.2 데이터 편향 영향  
    사용자 발화와 상담자 발화가 함께 입력되며 정서 강도가 완화되는 문제를 분석한다.  
    7.3 스타일 축 설계 한계  
    현재 스타일 축이 지나치게 공손함과 부드러움 중심으로 구성되어 있음을 지적한다.  
    7.4 프롬프트 평균화 문제  
    style summary와 tags가 평균적 상담형 말투를 재강화하는 구조적 문제를 설명한다.  
    7.5 연구 의의  
    최종 성능보다도 감정 동역학과 스타일을 분리해 분석 가능한 연구 프레임워크를 제시했다는 점의 의미를 정리한다.
    
8. 결론 및 향후 연구  
    8.1 결론  
    EmoNet이 감정 응답 생성의 내부 단계를 분리 가능한 구조로 제시했다는 점을 요약한다.  
    8.2 한계  
    z -> s 성능 부족, 인간 평가 부재, 스타일 축 편향, 고정 통계 인코더의 한계를 정리한다.  
    8.3 향후 연구  
    화자 역할 분리, raw affect 축 추가, 학습형 branch encoder 도입, baseline 및 human evaluation 확장을 제안한다.
